Create a `.brand.yaml` brand configuration file for this project through guided setup.

## Process

1. **Check for existing config**
   First check if `.brand.yaml` already exists in the project root. If it does, ask if user wants to update it or start fresh.

2. **Gather brand information** using AskUserQuestion tool

   Ask about visual identity:
   - Brand name and tagline
   - Primary color (main brand color for CTAs, links)
   - Secondary and accent colors
   - Whether this is a light or dark themed application
   - Typography preferences (display font, body font)

   Ask about brand voice:
   - 3-5 brand personality attributes (e.g., friendly, professional, playful)
   - Tone description (e.g., "warm and encouraging" or "direct and technical")
   - Any words to prefer or avoid

3. **Generate configuration**
   Create a comprehensive `.brand.yaml` file including:
   - All color values with semantic names
   - Typography configuration with scale
   - Spacing system
   - Voice and vocabulary guidelines
   - Comments explaining each section

4. **Provide next steps**
   After creating the config:
   - Suggest running `/branding-toolkit:brand-generate` to create CSS/Tailwind files
   - Mention `/branding-toolkit:brand-docs` for documentation
   - Note that brand-auditor agent will now check for compliance

## Output Location

Write the configuration to `.brand.yaml` in the project root.

## Example Output Structure

```yaml
# Brand Configuration
# Generated by branding-toolkit

name: "Brand Name"
tagline: "Your tagline"

attributes:
  - attribute1
  - attribute2

colors:
  primary: "#hex"
  # ... more colors

typography:
  display: "Font Name"
  body: "Font Name"
  scale: 1.25

voice:
  tone: "description"
  vocabulary:
    prefer: [words]
    avoid: [words]
```
